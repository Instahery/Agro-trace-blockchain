# Changelog - Enterprise Knowledge Management System

All notable changes to this project will be documented in this file.

---

## [Unreleased] - 2026-07-06

### Added
- **CFO Persona Integration**: Added support for strategic financial and governance analysis by processing raw narrative text and agro-climate data through a customized CFO-lens system prompt.
- **Dynamic Database Architecture**: Implemented full support for `cfo_analysis` and deep `agro_climate_*` JSON schema distribution into the `ai_analyses` table column structures.
- **Multi-Report Strategy Handler**: Upgraded `generate_weekly_summary2.php` to handle dual-mode logic (`weekly` and `monthly`) seamlessly inside a single script using URL query control parameters (`?type=weekly` or `?type=monthly`).
- **Conditional Cache Strategy**: Automated isolated JSON caching file names (`testknowledge_weekly.json` and `testknowledge_monthly.json`) to prevent cross-over stream data overwrites.

### Changed
- **Dynamic Print Interface (`cetak_jurnal_html.php`)**: Rewrote the printing and PDF-generation layouts into an adaptive engine. The file now sniffs database cell states natively (`!empty($row['kolom'])`) and structures components without hardcoding fixed persona visibility.
- **Print Layout Visual Styling**: Updated section headers with corporate dark-blue styling nodes to accommodate professional financial, administrative, and scientific print-outs using semantic HTML tables.

### Fixed
- **HTTP 500 Server Error**: Resolved a critical PHP fatal error on line 177 caused by a broken string-comment layout within a `foreach` loop construct (`end/* ... */foreach;`).
- **Syntax Error Correction**: Patched a legacy variable misprint (`<?html` instead of `<?php`) in the `<title>` tag context.
- **Data Duplication Guard**: Fixed a variable clash on `$risks` decoding by clean-separating raw strings and loop arrays (`$risks_arr`), preventing unexpected 500 crash behaviors on valid JSON blocks.

### Code Style & Governance
- Strictly adhered to standard APA style guidelines for `scientific_references` formatting arrays.
- Enforced clean fallback states (`NULL` and empty array constants) across all dynamic frontend tables to prevent raw data leaks or broken borders during physical network failures.
